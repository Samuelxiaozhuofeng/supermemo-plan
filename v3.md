# SuperMemo Plan Core App v3 计划

## 当前职责分配审查

v2 已经把展示组件从 `App.tsx` 中拆出一部分，当前已有这些相对清晰的职责边界：

- `src/components/ActivityTable.tsx`：活动表格展示与单元格编辑。
- `src/components/Timeline.tsx`：时间轴展示。
- `src/components/DelayAnalysisPanel.tsx`：延误分析展示。
- `src/components/HistoryPanel.tsx`：历史记录列表展示。
- `src/components/AdjustPanel.tsx`：Adjust 预览与选择展示。
- `src/components/AlarmModal.tsx`：提醒弹窗展示。
- `src/exporters.ts`：Markdown / CSV / JSON 导出与 JSON 备份解析。
- `src/storage.ts`：localStorage 读取、保存、v1 到 v2 兼容迁移。
- `src/types.ts`：核心类型定义。
- `src/time.ts`：时间解析、格式化、当前时间。

但职责还没有分配到位，主要问题：

- `src/App.tsx` 仍有 633 行，同时负责 app 状态、页面模式、模板操作、今日执行、历史归档、Adjust、导入导出下载、提醒计时、通知权限、布局和大量事件处理。
- `src/schedule.ts` 仍有 434 行，同时负责排程算法、延误分析、执行命令、历史归档、拆分合并、Adjust、ID 生成和展示辅助。
- 组件已经拆出，但控制逻辑没有拆出；目前是“展示拆分了，业务编排还集中在 App”。
- `App.test.tsx` 只覆盖主流程烟测，缺少按 feature/hook 的测试。

v3 的第一目标是职责重分配，而不是先加大功能面。

---

## v3 目标

1. 把 `App.tsx` 压缩为应用装配层，只负责布局组合和 hook 调用。
2. 把 `schedule.ts` 拆成独立 domain modules，避免算法、命令、Adjust、归档混在一起。
3. 把模板、执行、历史、导入导出、提醒分别放到明确文件。
4. 为每个职责边界补测试，防止后续继续变成单文件膨胀。
5. 在职责拆分完成后，再实现少量 v3 功能：活动统计、列管理、提醒增强。

## P0：代码职责重构

### 1. 拆分 `App.tsx`

目标：`App.tsx` 控制在 120 行以内。

新增文件：

- `src/components/AppHeader.tsx`
  - 顶部标题、模板选择、新模板、创建今日副本。
- `src/components/StatusStrip.tsx`
  - 模板 / 今日执行 / 历史 tab。
  - 当前时间和运行中活动剩余时间。
- `src/components/PlanEditorPanel.tsx`
  - 计划名称、起点、总分钟。
  - 理想总量、可用时间摘要。
- `src/components/PlanControls.tsx`
  - 新增、删除、上下移动、拆分、合并、跳过、完成当前。
- `src/components/ExecutionControls.tsx`
  - Begin、Terminate 并归档、Adjust、通知权限。
- `src/components/ExportControls.tsx`

`App.tsx` 最终只保留：

- 计算 view model。
- 传 props 给 layout components。
- 渲染 `AlarmModal`。

## P2：SuperMemo 还原增强

### 1. Rigid 刚性时段

新增字段：

```ts
rigid?: boolean;
```

行为：

- 刚性活动优先占用自己的 `desiredMinutes`。
- 非刚性活动在剩余时间中按比例压缩。
- 如果刚性活动超过总时长，返回显式冲突，不静默压缩。

需要新增：

- `src/domain/constraints.ts`
- `src/domain/computeSchedule.test.ts` 中的 rigid tests。
- 表格中的锁定列。

### 2. Reset

新增操作：

- 清空执行字段。
- 可选择保留或清除 fixedStart。
- 重算计划。

放置位置：

- domain：`editing.ts`
- UI：`PlanControls.tsx`

### 3. Adjust from now / selected

新增：

- 从当前时间重新压缩后续活动。
- 从所选活动开始重新压缩。
- 操作前显示影响范围。

放置位置：

- domain：`adjust.ts` 或 `execution.ts`
- UI：`AdjustPanel.tsx` 扩展。

## 推荐实施顺序

1. 创建 `src/domain/`，把 `schedule.ts` 拆成 domain modules，并保持现有导出兼容。
2. 把 `createBlankTemplate`、`copyHistoryToToday` 从 `App.tsx` 移到 `domain/factories.ts`。
3. 创建 hooks，把 `App.tsx` 中的 state/action/timer/adjust/import-export 逻辑迁出。
4. 拆出 header、status strip、左侧编辑面板、控制按钮和导出按钮组件。
5. 迁移并扩展测试，确保行为无变化。
6. 删除或缩小旧 `schedule.ts`，只保留 re-export。
7. 实现活动关键词统计。
8. 实现列显示管理。
9. 实现提醒设置增强。
10. 最后再做 Rigid、Reset、Adjust from now / selected。

---

## v3 完成标准

- `App.tsx` 少于 120 行。
- `schedule.ts` 不再直接包含业务实现，或删除该文件改用 `domain/` imports。
- 单个 domain 文件少于 180 行。
- 单个组件文件少于 180 行。
- 所有新增功能都有对应测试。
- `npm test` 和 `npm run build` 通过。
- UI 行为与 v2 保持兼容。

---

## 暂不进入 v3 的内容

- 云同步。
- 登录系统。
- PWA。
- 日历双向同步。
- AI 自动排程。
- 多人协作。

这些会引入新的架构方向，应该等核心单机计划器职责拆干净后再评估。
